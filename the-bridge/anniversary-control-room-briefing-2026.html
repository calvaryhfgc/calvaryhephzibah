Context
Why this briefing exists
Control Room was built for a normal Sunday: worship → announcements → scripture → sermon → offering → close. The anniversary on 17 May breaks that mould. Communion happens early at the tables; a visiting praise leader ministers in song during the offering; Pastor Shade calls members by name in a Show of Appreciation; a full sit-down meal sits inside the service. Each of those moments wants a different kind of overlay support than Control Room currently offers. This document walks the running order item-by-item, names where the operator app holds up and where it doesn't, and proposes a focused set of extensions ranked by how much they'd matter on the day.

Extensions
What to build, ranked by impact on the day
Five focused additions that would take Control Room from "fine for a normal Sunday" to "ready for the anniversary." Ranked by what they'd unlock on 17 May. Pills show approximate scope and how soon they'd realistically land.
01

Show of Appreciation — named-person cards

The single biggest gap. Pastor Shade is calling 8–12 members by name in sequence. The visual needs to keep up: photo + name + role + a one-line honour, pushed by Scott as Pastor Shade speaks. This is the emotional heartbeat of the anniversary; the screen should match her tone, not lag behind it.

  • New data model: an "honour queue" — ordered list of person cards with name, role, honour line, photo URL
  • Operator panel: scrollable list, tap to push, auto-advance toggle for quick succession
  • Overlay template: full-bleed warm anniversary treatment, photo dominant, name in Anton, honour line in Instrument Serif italic, brand red eyebrow
  • Preview pane so Scott sees what's next before he pushes
Scope: medium Build this week
02

Lower-thirds (everyone, everywhere)

Anniversary has more named individuals on stage than any normal Sunday — Pastor Shade for Welcome and Show of Appreciation and Closing, Pastor Gbenga preaching, Pastor Kayode MC-ing, Evangelist Bola Paul ministering, visiting ministers praying. Every one of them wants a name on screen when they first appear. Today there is no lower-third mechanism — it has to be faked with fullscreen images.

  • New overlay region: bottom-left band, ~140px tall, brand styled
  • Person library in Supabase (name, role/title, optional photo)
  • Operator panel: search-as-you-type, tap to push, auto-hide after 8 seconds
  • Works alongside lyrics and scripture without conflicting (z-index resolved)
Scope: medium Build this week
03

Presentation timer + slide upload

Three 3-minute presentation slots, each one allocated to a department, each one at risk of overrunning. The MC needs visible support. And every department brings slides/photos that today have no clean way to display. Both gaps share infrastructure.

  • Countdown overlay (top-right corner, brand-styled, configurable duration — 3 min default)
  • Visual escalation: green → amber → red at thresholds
  • Drag-and-drop slide upload in the operator app — drops directly into Supabase Storage
  • Slide-by-slide push with thumbnail preview
Scope: medium Build this week
04

Photo gallery loop (for The Meal)

60–75 minutes of meal time is the longest single block of the day. Right now the screen either holds a static graphic or runs a single backdrop video. A rotating gallery of church-life photos — Pastor Shade's 60th, past anniversaries, the worship team, ministry moments — would turn this hour into a celebration of the church's history rather than dead screen time. People look up, see themselves, see each other, see the journey.

  • New entity: gallery (named, ordered list of image URLs, transition timing, music track optional)
  • Operator: pick a gallery, play / pause / next / skip
  • Overlay: smooth crossfade between images, brand-coloured frame, subtle "34 years" watermark
  • Reuses existing Supabase Storage from the backdrops feature
Scope: medium Build this week or next
05

Communion sequence + offering bank-details preset

Two preset overlay sequences that get used at every service and especially under pressure on the day. Communion (1 Cor 11:23–26 + bread instruction + wine instruction + closing) is a 4-step sequence Scott has to build manually every time. Offering bank details is the most-pushed graphic in Calvary's history and still lives as an ad-hoc image.

  • Communion preset: 4-step sequence, one-tap advance, customisable text
  • Offering preset: live-editable account name / sort / account / reference, brand styled
  • Both available as one-tap buttons in the Order of Service panel
Scope: small Build this week
If nothing changes by Sunday
Fallback plan
In the realistic case that not all extensions land before 17 May, this is how Scott runs the day with the tools he has today.

For the Show of Appreciation — pre-build a dozen fullscreen anniversary-styled name cards as JPGs ahead of the day (Pastor Shade gives the list Wednesday). Load them all into Graphics → Images, in the order Pastor Shade will call them. Scott pushes them sequentially as names are called. If the order changes live, Scott finds the right card from the list and pushes.

For lower-thirds — pre-build name PNGs for the five named individuals on the day (Pastor Shade, Pastor Gbenga, Pastor Kayode, Evangelist Bola Paul, and one generic "Visiting Minister" card). Push as fullscreen images for ~8 seconds when each first speaks, then return to live shot.

For presentation timing — Pastor Kayode (MC) holds a physical stopwatch and signals each presenter at the 2-minute mark. Scott has no role here.

For department slides — coordinators send slides as JPGs by Friday. Scott pre-loads them into Graphics → Images in groups labelled "Pres 1", "Pres 2", "Pres 3". On the day, opens the right group and pushes manually.

For the meal hour — set a single beautiful Calvary backdrop (Pastor Shade's 60th, or the 34th anniversary hero) and leave it for the duration. Loop a worship-instrumentals playlist for audio. Live the limitation.
