What is the overlay

Before you touch OBS, understand what we're plugging in.

The overlay is a transparent web page that shows lyrics, Bible verses, and (soon) graphics on top of your camera feed. Someone — you, Bolaji, or any team member with a PIN — opens the operator app on their phone, taps through slides, and the overlay updates live in under a second.

Your job in OBS is just to add the overlay as a layer. Once it's set up, you don't touch it again. The operator drives it from elsewhere.

Think of it like this: OBS handles the camera, audio, and the stream going out to YouTube. The overlay sits on top of the camera as a second layer — same as a lower-third graphic would. Someone else controls what appears; OBS just shows it.

Setting up in OBS

One-time setup. Takes about 5 minutes.

1

Open OBS and pick your scene

In OBS, choose the scene you use for Sunday service — the one with the camera feed already set up. You'll be adding the overlay as a new source on top of the camera.

2

Add a new Browser Source

In the Sources panel (usually bottom-left), click the + button and choose Browser.

When prompted for a name, type Calvary Overlay. Click OK.

3

Paste the overlay URL

A settings panel will open. In the URL field, paste this exact link:

Overlay URL https://calvaryhfgc.github.io/calvaryhephzibah/control-room/overlay.html?hud=off
Why ?hud=off: that tiny bit at the end hides the green "live" status pill in the corner. Without it, viewers on the stream would see a small status badge. Always include ?hud=off for the live service. Drop it when testing.
4

Set the width and height

Set these to match your stream resolution — for YouTube that's almost always:

Width1920
Height1080

If you stream in 720p, use 1280 × 720 instead. The overlay scales automatically — what matters is that the width and height match whatever your scene is set to.

5

Tick the right checkboxes

Further down in the same panel, find these options and set them exactly like this:

Control audio via OBSOff
Shutdown source when not visibleOff
Refresh browser when scene becomes activeOn
Why these matter:
— Shutdown when not visible OFF: if on, the overlay disconnects from Supabase between uses. When someone pushes a new slide, it takes 2-3 seconds to reconnect. Off = always connected, instant updates.
— Refresh when scene becomes active ON: means if you switch to a different scene and come back, the overlay refreshes. Good safety net if anything ever freezes.
6

Click OK and position it

Click OK to close the settings. The overlay now exists in your scene.

In the canvas (the preview window), drag the corners of the Calvary Overlay source so it covers the whole screen — exactly the same size as your camera feed. It should be invisible right now (transparent background), but that's correct.

7

Check the layer order

In the Sources panel, Calvary Overlay must be above your camera source. Drag it up if it's not.

Order matters: sources at the top of the list render on top of sources below them. Camera at the bottom, overlay on top. Anything that should appear above the overlay (like a permanent church logo bug) goes above it.

Save your scene collection. In OBS, go to Scene Collection → Export and save a backup. If anything ever gets messed up, you can reimport and be back to working in seconds.

Testing before service

Always test the overlay before going live. Takes 60 seconds.

1

Open the operator app on your phone

Go to:

Operator app https://calvaryhfgc.github.io/calvaryhephzibah/control-room/

Sign in with your PIN. You'll land in the app with today's set list pre-loaded.

2

Temporarily show the status pill

For testing only, change the OBS Browser Source URL to drop the ?hud=off bit. Just for now — so you can see the status indicator.

Testing URL (with HUD visible) https://calvaryhfgc.github.io/calvaryhephzibah/control-room/overlay.html

In OBS, right-click the Calvary Overlay source → Properties → change the URL → OK.

In the top-right corner of the OBS canvas you should now see a small green dot with the word LIVE. That's the overlay confirming it's talking to the database.

3

Push a slide from your phone

On your phone, tap any song in the set list → tap slide 1. Within about a second, the lyric should appear on the OBS preview.

Tap Next a few times. OBS updates each time.

Tap Blank. OBS goes transparent.

If all three work — the overlay is ready for service.

4

Switch back to the production URL

Before you go live, change the URL back to the one with ?hud=off. Otherwise the status pill will appear in the livestream.

What to do on Sunday morning

Your pre-service routine for the overlay.

1

Boot OBS and verify the overlay source is in the scene

Check the Sources panel — Calvary Overlay should be there, above the camera. If someone accidentally deleted it, follow the Setup section again.

2

Right-click the Calvary Overlay source and hit Refresh

This makes sure it loads fresh when you start the service. OBS Browser Sources sometimes go stale if the computer has been on for a long time.

3

Confirm the operator has signed in

Whoever is driving the overlay from their phone should open the operator app and confirm today's set list appears. If they see "No set list loaded" — that's a problem for Bolaji, not OBS.

4

Push one test slide before going live

Ask the operator to push any slide. Confirm it appears on your OBS preview. Then ask them to tap Blank.

If that loop works, you're good to start the stream.

During the service: you don't need to do anything with the overlay. The operator drives it. Your job is the camera, audio, and the stream. If the overlay ever freezes or misbehaves, right-click Calvary Overlay in Sources → Refresh. It'll be back in 2 seconds.

Running the overlay from multiple machines

The thing that trips most people up: your OBS does not need to be on the same computer — or even the same room — as whoever is pushing slides.

How it actually works

The overlay, the operator app, and OBS are three separate things. They don't talk to each other directly. They all talk to a shared database in the cloud (Supabase). That's why any of them can run anywhere — phone, laptop, a different church, a different country — as long as the device has internet.

Your OBS machine just reads the database. Whoever is pushing slides writes to the database. They never communicate directly.

What this means for you: you don't have to run the operator app yourself. You don't have to be near whoever's driving it. Your only job is making sure OBS has the overlay loaded and the stream is healthy.

The typical Sunday setup

A

You — OBS at the media desk

You focus on camera, audio, and the stream out to YouTube. The Calvary Overlay source is loaded but you never interact with it during service. Everything else is handled remotely.

B

A second operator — laptop or phone anywhere in the room

This person sits with the worship team, the preacher's notes, or just at the back with a laptop. They open the operator app, sign in with their PIN, and tap through slides during service.

Could be Bolaji, Tolu, Auntie Pauline, or any trained team member. As the church grows you can have multiple people rotating in this role.

C

A backup operator — on their phone as a safety net

Because multiple people can be signed in at once, Bolaji can keep the operator app open on his phone during service. If the main operator fumbles — wrong slide, stuck on the chorus, forgets to blank between sections — the backup can push the right thing from their phone without getting up.

Both operators see each other's pushes on their Now Playing card, so there's no confusion about what's on screen.

Things this unlocks

Remote training. Bolaji can train you on the overlay from home — he runs the operator app, you watch your OBS update live, over a Zoom screenshare. No need to be in the same building.
Dry runs during the week. On a Saturday evening you can test the full loop from your sofa. Open OBS, load the overlay, open the operator app in another tab, tap through Sunday's set list. If anything's off you find out before 11 AM on Sunday.
Disaster recovery. If the media desk computer dies mid-service, any laptop with OBS installed can pick up where it left off — load the overlay URL into a fresh OBS install and the overlay state is whatever it was when the old machine went down. The database remembers.
Multi-site in the future. If Calvary ever plants a second campus, the same control room can drive overlays in two buildings simultaneously. One set list, two OBS machines at two locations, identical output.

What this does NOT change

You still need the Calvary Overlay source in OBS. You still need to test before service. You still need internet on the OBS machine. The distributed model is about who can push slides — not about magic wifi-free overlay rendering.

One honest limitation: if the church's internet goes down, the operator app can still push slides from a phone on 4G — but OBS will be stuck on whatever the overlay last showed, because OBS can't read from Supabase without its own internet. Tether the OBS machine to a phone's mobile hotspot if the church wifi ever fails mid-service.

Troubleshooting

Things that might go wrong, in order of likelihood.

Lyrics appear but text is tiny

Your Browser Source width and height don't match your scene resolution. Open the source properties and set Width to match your canvas (usually 1920), Height to match (usually 1080). The overlay scales with its canvas.

Operator pushes a slide but OBS doesn't update

Right-click Calvary Overlay in Sources → Refresh. Nine times out of ten, that fixes it. The overlay has probably lost its realtime connection to the database (internet blip, computer sleep, OBS was minimised for too long).

If refresh doesn't fix it, ask the operator to sign out of the operator app and sign back in. That forces a fresh connection from their end.

I see a small green pill in the corner of the stream

Your overlay URL is missing ?hud=off. Right-click source → Properties → add ?hud=off to the end of the URL → OK.

The overlay is showing but it's a solid colour, not transparent

Unusual but possible — some older OBS versions had issues with transparent backgrounds. Check: OBS version 28 or higher is required. Help → About should tell you. If older, update OBS (free, from obsproject.com).

"Browser" isn't an option when I add a source

You're on a very old OBS or a build that was installed without browser support. The fix is to reinstall OBS from the official site. Always choose the Browser Source plugin during install — it's included by default in current versions.

Text appears but looks blurry or pixelated

Check that your Browser Source dimensions match your scene resolution exactly. 1920 × 1080 in a 1920 × 1080 scene is crisp. 1280 × 720 stretched into 1920 × 1080 looks blurry.

Everything worked yesterday but it's broken today

First: right-click the source → Refresh. Second: ask Bolaji whether the operator app is working on his end — if it isn't, the issue is at the database, not OBS. Third: if nothing else works, delete the Calvary Overlay source and add it again from scratch using this guide. Takes 3 minutes.

Still stuck? Screenshot what you're seeing and send it to Bolaji on WhatsApp. Describing the problem in words is always harder than showing it.
